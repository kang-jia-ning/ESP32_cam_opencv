## 问题分析

调用豆包API时出现超时错误：`HTTPSConnectionPool(host='ark.cn-beijing.volces.com', port=443): Read timed out. (read timeout=10)`

可能的原因：
1. 网络连接不稳定或速度慢
2. 超时时间设置过短（当前10秒）
3. API服务器响应慢
4. 请求参数导致处理时间过长（如max_completion_tokens设置为65535）

## 解决方案

### 1. 增加超时时间
将超时时间从10秒增加到30秒，给API更多响应时间

### 2. 实现重试机制
- 当请求超时时，自动重试2-3次
- 每次重试之间添加指数退避（如1秒、2秒、4秒）

### 3. 优化请求参数
- 减少max_completion_tokens值，从65535调整为更合理的值（如2000）
- 保持reasoning_effort为medium，平衡质量和速度

### 4. 增强错误处理
- 在超时发生时返回友好的错误信息
- 记录详细日志，便于调试
- 实现降级策略，超时后返回模拟回答

## 实施步骤

1. 修改`app/routes/main.py`中的`ask_question`函数
   - 增加超时时间到30秒
   - 实现重试机制
   - 优化请求参数
   - 增强错误处理

2. 测试修改后的API
   - 使用测试脚本验证超时问题是否解决
   - 模拟不同网络条件下的请求

3. 监控API性能
   - 添加日志记录API响应时间
   - 分析请求成功率和响应时间

## 预期效果

- 减少API调用超时错误
- 提高智能问答功能的可靠性
- 为视障用户提供更稳定的服务
